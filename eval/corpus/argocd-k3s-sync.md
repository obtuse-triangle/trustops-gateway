# ArgoCD와 K3s 동기화 문제 해결 가이드

## 이미지 태그 불일치

ArgoCD가 K3s 클러스터에 배포할 때 이미지 태그가 최신 상태로 반영되지 않는 경우가 자주 발생합니다.
원격 저장소의 이미지 태그가 변경되었는데도 ArgoCD가 이전 구성을 계속 바라보는 문제는
동기화 설정에서 `automated` 정책이 제대로 구성되지 않았을 때 나타납니다.
`spec.syncPolicy.automated.prune` 옵션을 활성화하고 `selfHeal`을 켜면
감지된 차이를 자동으로 해소할 수 있습니다.

## 저장소 URL 오류

컨테이너 이미지 저장소의 URL이 잘못 지정되면 ArgoCD가 이미지를 가져오지 못합니다.
레지스트리 주소를 정확하게 입력했는지 확인해야 합니다.
사설 레지스트리를 사용하는 경우 `imagePullSecrets` 설정이 올바르게 적용되었는지
점검하는 것이 중요합니다. K3s는 containerd를 사용하므로 `registries.yaml` 파일에
미러 설정이나 인증 정보를 추가해야 할 수도 있습니다.

## 인증 정보 누락

초기 설정에서 ArgoCD가 K3s API 서버에 접근하기 위한 인증 정보가 누락되면
동기화 자체가 실패합니다. ArgoCD는 클러스터 등록 시 제공된 kubeconfig로
K3s 클러스터에 연결합니다. 인증서가 만료되었거나 토큰이 유효하지 않은 경우
`argocd cluster list` 명령어로 연결 상태를 확인하고 재인증해야 합니다.

## 다각적 접근의 중요성

동기화 문제가 발생했을 때 하나의 원인만 의심하지 말고
이미지 태그, 저장소 URL, 인증 정보, 네트워크 연결을 종합적으로 점검해야 합니다.
단일 단서에 의존하면 실제 원인을 놓치기 쉽습니다.
